More Information

The above example is simplified, but you can also specify more options such as constraints for your properties. DataMapper supports a lot of different property types natively, and more through dm-types. An important thing to note is that every model must have a key in order to be valid. If a model has no key, there's no way to identify a resource and thus no way to update its persistent state within the backend datastore.

More Content

DataMapper is intended to have a lean and minimalistic core, which provides the minimum necessary features for an ORM. It's also designed to be easily extensible, so that everything you want in an ORM can be added in with a minimum of fuss. It does this through plugins, which provide everything from automatically updated timestamps to factories for generating DataMapper resources.
